# 前端路由配置问题修复说明

## 🔍 发现的问题

### 问题 1: Vite 代理配置错误 ❌

**文件**: `clients/vite.config.ts`

**错误配置**:
```typescript
proxy: {
  '/api': {
    target: 'http://localhost:8000',  // ❌ 错误！应该指向 Gateway 8080
    changeOrigin: true,
    rewrite: (path) => path.replace(/^\/api/, '')
  }
}
```

**修复后**:
```typescript
proxy: {
  '/api': {
    target: 'http://localhost:8080',  // ✅ 正确！指向 API Gateway
    changeOrigin: true,
    rewrite: (path) => path.replace(/^\/api/, '')
  }
}
```

### 问题 2: Dashboard 路由未配置 ⚠️

前端期望的路由：
- `/dashboard/stats`
- `/dashboard/activities`
- `/dashboard/recent-projects`
- `/dashboard/charts`

**但是**：这些路由在 API Gateway 中**没有配置**！

## 📋 Vite 代理工作原理

### 代理流程

```
前端页面 (localhost:3000)
    │
    │ 请求: GET /api/auth/profile
    ▼
Vite 开发服务器 (localhost:3000)
    │
    │ 匹配代理规则: /api/*
    │ 移除前缀: /api/auth/profile → /auth/profile
    │ 转发到: http://localhost:8080/auth/profile
    ▼
API Gateway (localhost:8080)
    │
    │ 匹配路由: /auth/profile
    │ 转发到: http://localhost:25698/auth/profile
    ▼
Auth Service (localhost:25698)
```

### 为什么需要 `/api` 前缀？

1. **开发环境**: Vite 代理通过 `/api` 前缀识别需要转发的请求
2. **生产环境**: Nginx/反向代理也通过 `/api` 识别 API 请求
3. **避免冲突**: 前端静态资源路径不会与 API 路径冲突

## 🎯 Dashboard 路由解决方案

### 方案 1: 添加 Dashboard 路由到 Gateway（推荐）

如果你有一个 dashboard service 或者 dashboard 数据来自某个现有服务（如 project-service），需要在 `init_gateway_routes.py` 中添加路由：

```python
# 在 ROUTES 列表中添加
{
    "service_name": "project-service",  # 或 "dashboard-service"
    "path_pattern": "/dashboard/stats",
    "target_url": "/dashboard/stats",
    "method": "GET",
    "requires_auth": True,
    "rate_limit_rpm": 500,
    "priority": 8,
    "description": "获取仪表板统计"
},
{
    "service_name": "project-service",
    "path_pattern": "/dashboard/activities",
    "target_url": "/dashboard/activities",
    "method": "GET",
    "requires_auth": True,
    "rate_limit_rpm": 500,
    "priority": 8,
    "description": "获取活动列表"
},
{
    "service_name": "project-service",
    "path_pattern": "/dashboard/recent-projects",
    "target_url": "/dashboard/recent-projects",
    "method": "GET",
    "requires_auth": True,
    "rate_limit_rpm": 500,
    "priority": 8,
    "description": "获取最近项目"
},
{
    "service_name": "project-service",
    "path_pattern": "/dashboard/charts",
    "target_url": "/dashboard/charts",
    "method": "GET",
    "requires_auth": True,
    "rate_limit_rpm": 500,
    "priority": 8,
    "description": "获取图表数据"
},
# 通配符路由
{
    "service_name": "project-service",
    "path_pattern": "/dashboard/*",
    "target_url": "/dashboard/*",
    "method": "ANY",
    "requires_auth": True,
    "rate_limit_rpm": 1000,
    "priority": 1,
    "description": "Dashboard通配路由"
},
```

然后运行：
```bash
cd api_gateway_service
python init_gateway_routes.py
```

### 方案 2: Mock 数据（临时）

如果后端还没有实现 dashboard 接口，可以在前端临时 mock：

```typescript
// src/services/api/dashboardApi.ts
export const dashboardApi = {
  async getStats() {
    // 临时返回 mock 数据
    return {
      totalProjects: 10,
      activeUsers: 50,
      totalTasks: 100
    };
  }
};
```

### 方案 3: 创建 Dashboard Service

如果需要专门的 dashboard 服务，可以：

1. 创建 `dashboard_service` 目录
2. 实现 dashboard 接口
3. 启动服务（如 port 25710）
4. 在 Gateway 中注册路由
